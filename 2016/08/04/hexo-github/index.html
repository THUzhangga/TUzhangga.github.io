Hexo + Github Pages搭建静态博客的艰辛之路

本来是想着利用github的账号来给自己设置一个好看的主页，但是无奈对设计一窍不通，套用现成的bootstrap模板还得修改，改半天也达不到自己想要的效果。后来就想着不如搭个静态博客，方便而且模板也很简洁。谁知第一次搭的时候被ssh协议弄蒙了（其实现在也不太清楚），没有成功就搁置下来，到今天发现有一个能写东西的地方很不错，
不是为了跟谁炫耀自己做的主页好看，纯属是技术+搞笑~~

  1. 创建github账户
    首先你要注册一个github账户，虽然网站是英文的，但是做技术的我们必须也得克服。在自己的账户下创建一个名为username.github.io的仓库。
  2. 安装Git和node.js
    此处不再赘述详细过程，但是要注意，node要安装在全局环境下，我们得用npm的命令。
  3. 使用Hexo建站
    在此之前必须保证电脑已经安装好Git和Node.js
    • 首先安装Hexo
      $ npm install -g hexo(-g 表示在全局环境下安装Hexo)
      hexo -v 可以查询当前hexo版本
    • 创建本地文件
      在你喜欢的目录下创建一个文件夹，比如说:d/blog，在此处用Git bash打开。
      依次输入：$ hexo init 初始化文件夹
      $ npm install  下载所有的依赖包
$ hexo generate  编译生成待发布的博客文件。
$ hexo server  在本地访问查看建好的博客
  4. 配置SSH Keys
    • 设置用户名称和邮件地址
      $ git config –global user.name “username”
      $ git config –global user.email “username@example.com”
    • 检查电脑上现有的ssh key
      $ cd ~/.ssh 检查本机的ssh密钥(如果提示：No such file or directory 说明你是第一次使用git。)
    • 生成新的SSH Keys
      $ ssh-keygen -t rsa -C “邮件地址@youremail.com”
      Generating public/private rsa key pair.
      Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>
      然后系统会要你输入密码：(直接回车即可)
      Enter passphrase (empty for no passphrase):<输入加密串>
      Enter same passphrase again:<再次输入加密串>
      最后看到这样的界面，就成功设置ssh key了：
    • 在本机设置SSH Key之后，需要添加到GitHub上，以完成SSH链接的设置。
      1.打开本地C:\Documents and Settings\Administrator.ssh\id_rsa.pub文件。此文件里面内容为刚才生成人密钥。如果看不到这个文件，你需要设置显示隐藏文件。准确的复制这个文件的内容，才能保证设置的成功。
      2.登陆github系统。点击右上角的 Account Settings—>SSH Public keys —> add another public keys
      3.把你本地生成的密钥复制到里面（key文本框中）， 点击 add key 就ok了
    • 测试
      可以输入下面的命令，看看设置是否成功，git@github.com的部分不要修改：
      $ ssh -T git@github.com
      如果是下面的反馈：
      The authenticity of host ‘github.com (207.97.227.239)’ can’t be established.
      RSA key fingerprint is 16:27:ac:a5:76:28:2d:36:63:1b:56:4d:eb:df:a6:48.
      Are you sure you want to continue connecting (yes/no)?
      不要紧张，输入yes就好，然后会看到：
      Hi 你的用户名! You’ve successfully authenticated, but GitHub does not provide shell access.
  5. 将本地Hexo部署到Github上
    • 编辑d/blog下的_config.yml
      deploy:
      type: git
      repo: 对应仓库的SSH地址（可以在GitHub对应的仓库中复制）
      branch: 分支（User Pages为master，Project Pages为gh-pages）
    • 为了能够使Hexo部署到GitHub上，需要安装一个插件：
      $ npm install hexo-deployer-git –save
    • 然后，执行下列指令即可完成部署：
      $ hexo generate
      $ hexo deploy
  6. 如果想要使用其他主题，使用gitclone将别人的主题拷贝到D:\Hexo\themes下，然后将_config.yml中的theme: landscape改为对应的主题名字。
